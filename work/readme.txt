Компрессор реализован полностью, декомпрессор частично (не вынесен для отдельного вызова). 
В сжатый файл записываются заголовок и бинарные данные кодов. 
Далее сжатый файл независимо читается, начиная с заголовка, и данные декодируются обратно.
В итоге программой создаются 3 файла (кроме исходного): 
1. Файл с расширением ".f101" содержит сами коды.
2. Файл с расширением ".comp" содержит сжатые данные с заголовком, является финальным для компрессора.
3. Файл с префиксом "_dec" и расширением исходного файла является восстановленным в результате вызова декомпрессора.

Имя исходного файла подается в качестве 1го аргумента командной строки.
